How it Works

Get your OData API up and running in just a few simple steps.

1. Configure

Edit config.json to set up your database connection (MySQL, PostgreSQL, MSSQL) and server settings.

2. Connect

Use the GUI to select your database dialect and establish a secure connection via SSH tunnel if needed.

3. Start Server

Launch the Waitress server with a single click. Access your API at localhost or configure HTTPS proxy.

4. Query

Use OData syntax to query your data: $select, $filter, $top, $orderby. Insert and update via JSON.

Frequently Asked Questions

Find answers to common questions about PulseConnector.

PulseConnector supports MySQL, MariaDB, PostgreSQL, and Microsoft SQL Server. You can easily switch between databases by changing the active_dialect in your config.json file.
PulseConnector provides built-in SSH tunnels for secure connectivity. All tunnels use encrypted SSH connections, so you don't need to open public ports on your local network. Additionally, JWT-based authentication protects your OData endpoints.
PulseConnector supports $select, $filter, $top, and $orderby for queries. Filter operators include eq, ne, gt, lt, ge, le, and like. You can also perform INSERT (POST), full UPDATE (PUT), and partial UPDATE (PATCH) operations via JSON.
PulseConnector is compatible with Windows and Linux x64 systems. The package installation includes embedded libraries, and the lightweight Waitress backend ensures consistent performance across both platforms.
Yes, PulseConnector is free for non-commercial use. Commercial use requires prior written authorization from the author. Contact us for commercial licensing options.
