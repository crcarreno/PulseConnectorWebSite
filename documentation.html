Getting Started

PulseConnector is a lightweight Python database connector that transforms your data into a ready-to-use OData API in minutes.

General Architecture

Test Local Server

Once installed, you can test your server at the following endpoints:

  • Local: http://localhost:4545/status
  • Remote: https://localhost:5000/status

Configuration

PulseConnector uses a config.json file for all settings. Below is a complete example:

{
    "ssh": {
        "enabled": true,
        "vps_host": "0.0.0.0",
        "vps_user": "user",
        "vps_pass": "root",
        "remote_db_port": 3306,
        "local_bind_port": 5001
    },
    "db_mysql": {
        "dialect": "mysql",
        "host": "127.0.0.1",
        "port": 3306,
        "user": "root",
        "pass": "root",
        "database": "northwind"
    },
    "db_postgres": {
        "dialect": "postgres",
        "host": "127.0.0.1",
        "port": 5432,
        "user": "postgres",
        "pass": "postgres",
        "database": "postgres"
    },
    "db_mssql": {
        "dialect": "mssql",
        "host": "127.0.0.1",
        "port": 1433,
        "user": "sa",
        "pass": "admin.1234",
        "database": "dbTest"
    },
    "server": {
        "host": "127.0.0.1",
        "port": 5000,
        "protocol": "http",
        "threads": 8,
        "connection_limit": 100,
        "backlog": 512,
        "channel_timeout": 60,
        "cleanup_interval": 30
    },
    "odata": {
        "pool_size": 20,
        "max_overflow": 10,
        "pool_timeout": 10,
        "pool_recycle": 1800,
        "pool_pre_ping": true
    },
    "active_dialect": "mssql"
}

hostDatabase server address

passDatabase password

Authentication

PulseConnector uses JWT-based authentication to protect your OData endpoints.

Generate Token

Send a POST request to the login endpoint:

POST https://localhost:5000/login

{
    "username": "admin",
    "password": "admin"
}

Generate Token using Postman

Use Token in Requests

Include the token in the Authorization header:

Authorization: Bearer <your-token-here>

Using Bearer Token in requests

OData Queries

PulseConnector supports standard OData query parameters.

$select - Select Columns

Choose specific columns to return:

GET /odata/customers?$select=id,company

$filter - Filter Results

Available filter operators:

Operator Description Example
eqEqual$filter=status eq 'active'
neNot equal$filter=status ne 'deleted'
gtGreater than$filter=price gt 10
ltLess than$filter=price lt 100
geGreater or equal$filter=quantity ge 5
leLess or equal$filter=quantity le 50
likePattern match$filter=name like 'John%'
 
# Filter examples
/odata/products?$filter=price gt 10
/odata/users?$filter=active eq true
/odata/orders?$select=id,ship_name&$filter=ship_name like 'Karen%'

$top - Limit Results

GET /odata/logs?$top=50

$orderby - Sort Results

# Descending order
/odata/products?$orderby=price desc

# Ascending order
/odata/users?$orderby=name asc

Combined Query

GET /odata/products?$select=id,name,price&$filter=price gt 20&$orderby=price desc&$top=10

Main Components

  • JsonEditor: Visual editor of config.json with an expanded, editable tree structure.
  • SshTunnelManager: Handles SSH forward and reverse tunnels from Python.
  • build_connection_string(cfg): Builds the connection string according to the selected dialect.
  • PySide6 GUI: Dynamic interface with ComboBox populated from db_* sections in JSON.
  • OData Service: Embedded OData service ready to expose your local database data.

Security

Network Architecture with SSH Tunnels

  • All tunnels use encrypted SSH connections.
  • No need to open public ports on your local network.
  • The reverse tunnel allows only the authorized VPS to access your OData.
  • JWT-based authentication with Bearer tokens.
  • Token expiration handling for secure sessions.
  • Secure token signing for protected endpoints.
